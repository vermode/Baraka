# API Contract

The shared agreement between the frontend and backend. Defines every API endpoint exactly once.

## Folders

| Folder | What it is | Edit it? |
|---|---|---|
| `spec/` | The `openapi.yaml` file — **the source of truth** for all API endpoints. | ✅ YES |
| `zod/` | Auto-generated Zod schemas (used by the backend to validate request bodies). | ❌ NEVER — regenerated by `pnpm codegen` |
| `react-client/` | Auto-generated React Query hooks (used by the frontend to call the API). | ❌ NEVER — regenerated by `pnpm codegen` |

## Workflow

1. Edit `spec/openapi.yaml` to describe a new endpoint or change an existing one.
2. Run `pnpm codegen` from the repo root.
3. Both `zod/` and `react-client/` regenerate.
4. The backend uses Zod schemas from `@workspace/api-zod`.
5. The frontend uses React Query hooks from `@workspace/api-client-react`.
6. If you change the API in a breaking way, both sides will get TypeScript errors and force you to update them — that's the whole point.

## Tools

- **orval 8** does the code generation. Config: `spec/orval.config.ts`.
- **Zod** is used for runtime validation. Same library on both sides.
